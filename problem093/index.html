Problem 93 (Arithmetic expressions)

Solution detail

Brute-force is the easiest way to approach this problem - the question is how to implement it efficiently and ergonomically!

One approach is to consider what targets can be reached from particular sets of digits. Given a particular set of digits, all possible targets are obtained by considering all the ways of splitting that set into two non-empty subsets, taking all possible targets from the two subsets and combining them with every possible operation.

This framing of the problem leads to an iterative solution whereby the base case consists of the singleton sets containing a single digit - for them, the only possible target is the number itself.

From there, iterate over the possible sets of digits in order of increasing size, combining the answers to previous sets in all possible combinations.

Once all targets have been calculated, it's a simple matter of filtering to integer answers and doing a bit of counting.
